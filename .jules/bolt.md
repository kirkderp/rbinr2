## 2024-05-15 - Use sort_unstable for primitive types
**Learning:** In Rust, `.sort()` uses a stable sort which allocates memory (O(n) auxiliary space). When sorting primitive types like `String`, `PathBuf`, or integers where elements are distinguishable only by their sorted value, the stability of the sort does not matter. The codebase was using `.sort()` for `Vec<String>` and `Vec<PathBuf>`, leading to unnecessary memory allocations and slightly worse performance compared to `.sort_unstable()` (which uses pattern-defeating quicksort with O(log n) auxiliary space).
**Action:** Use `.sort_unstable()` by default for sorting `Vec<T>` where `T` is a primitive or standard library type (like `String` or `PathBuf`) unless stable sorting is explicitly required (e.g., when sorting complex structs by a specific field where original order needs to be maintained).
