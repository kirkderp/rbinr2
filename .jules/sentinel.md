## 2024-05-25 - Inconsistent r2pipe Command Sanitization Allows Shell Escapes
**Vulnerability:** Multiple validation functions (`validate_search_pattern`, `validate_type_arg`, `validate_classname`, `validate_query_command`, etc.) for user inputs meant for r2 commands failed to completely block internal shell-like operators supported by r2 (such as `<`, `>`, `&`, `!`). This inconsistent filtering allowed users to potentially achieve arbitrary file writes, data exfiltration, or execute arbitrary shell commands (command injection).
**Learning:** `r2` has an internal shell engine that interprets certain operators (like output redirection `>` and subshell execution `!`) before running commands. Standard string parameter validation must exhaustively block all such shell metacharacters, rather than just common command separators like `;` or `\n`, to prevent malicious users from breaking out of the intended r2 context.
**Prevention:** Always maintain a consistent, comprehensive denylist of r2 shell metacharacters (`>`, `<`, `&`, `!`, `;`, `|`, `` ` ``, `\n`, `\r`) when validating inputs destined for r2pipe commands. Better yet, create and use a single, centralized input sanitization function for all such inputs to ensure consistency.
